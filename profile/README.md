## 📌 프로젝트 소개 (About)

취업 정보는 넘쳐나지만, 정작** “나에게 맞는 기업이 무엇인지”** 아는 것은 어렵습니다.
기업이 중요하게 생각하는 인재상은 매년 바뀌고, 구직자 개인의 시선과 기업이 보는 시선 사이에는 큰 정보 비대칭이 존재합니다.

**고용희(HireHee)**는 
📌 기업의 인재상 +
📌 기업에 대한 매경의 데이터 +
📌 사용자 성향 분석 +
📌 AI 기반 기업 관점 역매칭
을 결합하여, 기업 입장에서 봤을 때 사용자가 어떤 이미지로 보이는지를 분석합니다.

<br>

## 💡 기획 배경 (Background)

취업 시장에는 구직자와 기업 간의 **'미스매치(Mismatch)'**가 존재합니다.

* **📉 스펙 중심 준비의 한계**: 인사담당자의 **79.7%**는 스펙이 완벽해도 인재상이 맞지 않으면 탈락시킨다고 응답했습니다.
* **🧩 정보의 비대칭**: 구직자들은 최신 트렌드가 반영된 인재상 정보를 찾기 어렵고, '내가 보는 나'와 '기업이 보는 나' 사이의 괴리를 파악하기 힘듭니다.
* **⚖️ 직무 적합성 중요**: 구직자들에게 **직무 적합성(39.6%)**은 연봉, 워라밸과 함께 회사를 선택하는 3대 필수 조건이 되었습니다.

**👉 고용희는 이러한 문제를 해결하기 위해 '기업 관점의 역매칭 서비스'를 제안합니다.**

<br>

## ✨ 핵심 기능 (Key Features)

### 1. ⏱️ Easy – 초간단 성향 분석
복잡한 검사 없이 단 **5개의 진단 질문**으로 성향을 파악합니다.
소요 시간: 약 **2분**

### 2. 📰 Objective – 객관적 데이터 기반 분석
'뇌피셜'이나 단순 홍보성 멘트가 아닌, **매일경제(Maeil Business Newspaper) 기사 DB**를 기반으로 기업의 실제 인재상을 분석합니다.

### 3. 🔄 Matching – 기업 관점의 역매칭
지원자의 주관적 판단 오류를 해결하기 위해 **기업의 시선**에서 사용자를 진단하고 가장 적합한 기업을 추천합니다.

## 🛠️ 기술 스택 및 흐름

### 📱 Frontend — React Native
- 모바일 UI 구현  
- 성향 테스트 및 기업 추천 결과 화면 제공  

### ⚙️ Backend — FastAPI
- 사용자 응답 수집  
- AI 분석 엔드포인트 제공  
- 기업 인재상 스코어링 로직 처리  

### 🧠 AI Layer (Docker 내부 구성)
- **COT 기반 성향 분석 모델**  
  - 사용자 응답 → 인재상 태그로 구조화  
- **Ollama + LLM**  
  - 기업 기사 기반 패턴 분석  
  - 인재상 키워드 자동 추출  
- **Stable Diffusion + LoRA**  
  - 사용자 성향을 시각적 이미지로 표현  
  - 추천 결과를 더 직관적으로 전달  

### 📂 데이터 — BigKinds (매일경제 기사 DB)
- 최근 5년 기사 기반  
- 기업별 핵심 키워드 벡터 생성 및 분석
